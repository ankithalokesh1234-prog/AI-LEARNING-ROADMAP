# Day 4 - Functions & Modules

# Functions
A function is a reusable block of code used to perform a specific task.

# Function Call
A function must be called to execute the code inside it.

# Parameters
Parameters are variables defined in a function definition.

# Arguments
Arguments are the actual values passed to a function.

# Positional Arguments
Values are assigned to parameters based on their position.

# Keyword Arguments
Values are assigned using parameter names.

# Default Arguments
A default value is used when no argument is provided.

# *args
Used to accept multiple positional arguments.

# **kwargs
Used to accept multiple keyword arguments.

# Return Statement
Returns a value from a function to the caller.

# Local Scope
Variables created inside a function can only be accessed within that function.

# Global Scope
Variables created outside a function can be accessed throughout the program.

# LEGB Rule
Python searches variables in the order Local, Enclosing, Global, and Built-in.

# Modules
A module is a Python file that contains reusable code.

# Import Statement
Used to access functions and features from modules.

# Math Module
Provides mathematical functions such as square root and factorial.

# Random Module
Used to generate random numbers and random selections.

# OS Module
Provides functions for interacting with the operating system.

# Datetime Module
Used to work with dates and time.

# Mini Projects
- Square & Temperature Converter
- Random Password Generator
- Student Result System

# Key Learning
Functions make code reusable and organized, while modules help structure programs and reuse code efficiently.
